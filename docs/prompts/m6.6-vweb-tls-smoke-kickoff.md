# VSL/MSL M6.6 — `vweb`: the §9 TLS smoke test (capstone close)

M6.1–M6.5 built the full vertical: the HTTP codec (`STDHTTPMSG`), the server
framework (`STDHTTPD`), the listener (`VWEBL`/`VWEBIO`), the first route
(`VWEBR` FHIR `GET /Patient/:id`), and auth (`VWEBA` Bearer→DUZ/#200, 401/403).
**M6.6 closes the capstone**: prove the *whole* stack end-to-end — a real client
makes an authenticated FHIR request and gets a Patient back — and run the §9 TLS
smoke test, then a clean KIDS install→verify→back-out on both engines.

## Where we are (read, don't re-derive)

- **M6.5 DONE** (see `docs/memory/m6.5-vweb-auth.md` + shared
  `docs/memory/m6.5-auth-stack.md`): a protected `GET /Patient/:id`, dual-engine
  green incl. live vehu/foia. The auth stack spans 3 repos (m-stdlib STDJWT /
  v-stdlib VSLSEC `$$bySecid` / v-web VWEBA).
- **The cross-repo OWED from M6.5 must close FIRST** (it gates a real install):
  (USER) tag **MSL v0.13.0** (STDJWT) → merge v-stdlib `m6.5-secid-binding` →
  v-web **repin to v0.13.0 + add the STDJWT seam** (`make pin`) → merge
  `m6.5-auth`. Until MSL v0.13.0 is tagged, STDJWT is consumed-but-unpinned.
- **TLS is the known infra gap (M2.T2, still blocked):** no server cert and
  `XU*8.0*787` (the YDB/IRIS TLS Kernel patch) absent. `VWEBIO`'s TLS path is
  **bound + loud** (`,U-VWEB-NOTLS,`), never a silent plaintext fallback. So the
  literal HTTPS handshake is **likely still blocked** — scope M6.6 as
  HTTP-first-loud (the established posture) and gate the real-TLS leg behind
  cert+patch availability, soft-skipping loudly when absent.

## THIS SESSION (M6.6) — the end-to-end smoke

1. **Full authenticated vertical over a real socket** (both bare engines + live):
   a client opens a socket, sends `GET /Patient/<dfn>` **with a valid Bearer
   token**, and reads back a 200 FHIR Patient (or 401 without the token / 403
   unprovisioned). This composes M6.3's `$$serveConn^VWEBL` + M6.5's VWEBA over
   the wire (M6.5 proved the 401-over-socket; M6.6 adds the *authenticated* 200
   over the wire — needs the JWT key in `^VWEB("rtcfg")`/XPAR + a resolvable
   subject, the M6.5 live pattern).
2. **§9 TLS smoke (gap-loud):** attempt the TLS bind via `VWEBIO`/`VWEBCFG`
   (`VWEB TLS SERVER CONFIG`); when cert+`XU*8.0*787` are present, do a real
   HTTPS round-trip; when absent, assert the loud `,U-VWEB-NOTLS,` and
   soft-skip (document the infra block — do NOT fake a handshake).
3. **KIDS install→verify→back-out** of `VWEB*1.0*3` on vehu + foia over the
   driver (the M0a lifecycle). Note the known v-pkg gap: uninstall is routine-
   only (#9.7/#9.6) and would orphan the XPAR param defs + the `VWEB LISTENER`
   option — either extend v-pkg uninstall to cover #8989.51/#19, or document the
   manual param/option cleanup. (This is the M6.3/M6.4 residual finally closing.)
