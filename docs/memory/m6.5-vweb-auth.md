# M6.5 DONE ‚Äî VWEBA auth middleware (Bearer/JWT ‚Üí DUZ/#200)

M6.4 mounted an OPEN `GET /Patient/:id`; M6.5 closes it. `VWEBA` is an STDHTTPD
middleware (registered ahead of the handlers) that authenticates a Bearer token
and binds it to a VistA principal. The capstone of the cross-repo auth stack ‚Äî
see m-stdlib [[m6.5-stdjwt]] (token validation, `m`) and v-stdlib
`m6.5-vslsec-secid-binding` (the #200 binding, `v`).

## The architecture (grounded; the WHY)

Deep research (vdocs gold corpus + VA IAM/PIV/SMART standards) established: **an
app validates a TOKEN, never the PIV card.** VA auth = PIV+PIN ‚Üí AD ‚Üí IAM
SSOi/STS ‚Üí a signed token (SAML for staff, OAuth/JWT for APIs); the relying party
validates the token + maps it to a local user (Kernel itself does this via
XUSAML‚ÜíXUESSO2, SecID‚Üí#200). VWEBA is that relying party. Lowest approval barrier
= delegate to the already-ATO'd VA IAM/SSOi + be a SMART-on-FHIR ¬ß170.315(g)(10)
resource server (inherit IA controls), NOT roll custom auth. The work splits on
the m/v waterline: token validation = `m` (STDJWT); #200 binding = `v` (VSLSEC).
(User decision 2026-06-17: full stack, cross-repo, JWT/RFC-9068 default.)

## What shipped (VWEBA)

- **`authn^VWEBA(.REQ,.RSP,.CTX)`** ‚Äî the middleware. Open-path allow-list
  (`/healthz`); extract Bearer (`$$bearer`); validate via the **provider seam**
  `$$validate` (M6.5 wires the **jwt** provider = `$$verify^STDJWT`; introspection
  /SAML are future providers behind the same `VWEB AUTH PROVIDER` dispatch); bind
  subject‚Üí#200 IEN via `$$bind` (`secid`=`$$bySecid^VSLSEC` default / `ien`=the
  subject IS a #200 IEN via `$$user^VSLSEC`); set `DUZ`; **401** unauthenticated
  (+`WWW-Authenticate: Bearer` + FHIR OperationOutcome), **403** authenticated-
  but-unprovisioned.
- **`register^VWEBA(.SRV)`** ‚Äî `do use^STDHTTPD(.SRV,"authn^VWEBA")`; called from
  `routes^VWEBR`. `getPatient` stays byte-for-byte auth-agnostic.
- VWEBA makes **no direct VistA call** (all via STD*/VSL*); the two L4 symbols it
  $text-PROBES (`GET^XPAR`, `GET1^DIQ`) carry `@icr` tags (2263 / notional DBS)
  to satisfy the no-direct-L4 gate ‚Äî they record the dependency, never invoke.

## Non-obvious decisions / gotchas

- **Config via a runtime cache.** `$$cfg` reads `^VWEB("rtcfg",<param>)` FIRST
  (listener-warmable; **tests set it directly so the whole chain runs with NO
  XPAR and no KIDS install**), then XPAR (VWEBCFG), then a default. Params added
  to KIDS as XPAR defs (`VWEB AUTH JWT KEY`/`PROVIDER`/`ISSUER`/`AUDIENCE`/
  `SUBJECT CLAIM`/`IDENTITY MAP`).
- **FAIL-CLOSED**: an empty/unconfigured signing key rejects ALL tokens (401),
  never silent-open. (`tUnconfiguredKeyFailsClosed`.)
- **DUZ is SET, not killed.** authn sets `DUZ`=bound IEN on every authenticated
  request before the handler runs; a failed auth short-circuits (`CTX("stop")`)
  BEFORE the handler, so a stale keep-alive DUZ never reaches a handler. No
  `kill DUZ` (it would break the bind's own `$$user` context).
- **Binding is $text-gated** (`$$hasVista`=`$text(GET1^DIQ)`): on a bare engine a
  cryptographically-verified request passes through with DUZ unresolved (bare
  handlers touch no FileMan); the #200 binding is a live-VistA operation.
- **Test-separation:** registering the middleware in `routes^VWEBR` protected
  `/Patient/:id`, breaking VWEBRTST's open-route tests. Fix: VWEBRTST's handler/
  routing tests use a `routesNoAuth` helper (`do routes^VWEBR(.SRV) kill
  SRV("mw")`) ‚Äî the real table with the middleware chain stripped. End-to-end
  auth on the route is VWEBATST's job.
- **TWO test gotchas burned here:** (1) a suite-local `if $$hasVista()` call when
  no `hasVista` label exists in the suite ‚Üí undefined-label ‚Üí silent **0/0
  abort** (the runner swallows $ZSTATUS). Use the label that EXISTS
  (`$$hasFileMan`). (2) the "bare signing" security tests must SKIP on a live
  engine (`if $$hasFileMan() quit`) ‚Äî else binding kicks in and a sub=1 token
  with the default `secid` map ‚Üí `bySecid("1")`‚Üí403, not the expected 400.
- **STDCRYPTO availability:** token-signing tests guard on `$$available^STDCRYPTO`
  ‚Äî present on the bare test engines AND on vehu/foia (so the full e2e runs live
  on both); the no-signing binding test (`tByIenLive`) proves the binding even
  without crypto.
