# M6.6 DONE — the §9 end-to-end smoke (M6 capstone CLOSED)

M6.1–M6.5 each proved one layer (codec / framework / listener / route / auth);
**M6.6 proves the WHOLE vertical composed over a real socket, both engines** —
the reason the seam was built. It adds **no production surface**: it is
composition + verification, driving the already-green VWEB*/VSL*/STD* stack
through one front door. New suite **`tests/VWEBE2ETST.m`** (the capstone smoke).

## The capstone gap this closed

M6.5 proved **401-over-socket** (`t401OverSocket`) and the **authenticated 200
via `$$dispatch`** (fake transport); M6.4 proved the **200-over-socket with auth
STRIPPED** (`routesNoAuth`). The one leg never tested end-to-end was the
**authenticated 200 over a real socket** — auth middleware + #200 binding +
VSLFS #2 read + STDJSON serialization, all at once on the wire. `VWEBE2ETST`
`tAuthenticatedPatient200OverSocket` is exactly that, and it is **green live on
vehu** (a real Bearer token → DUZ → FileMan read → 200 FHIR Patient byte-read
back off the socket).

## Non-obvious gotchas (burned here, fixed)

- **STDHTTPD route table is INDEXED, not path-keyed.** `routes^VWEBR` builds
  `SRV("route")=count`, `SRV("route",n,"pattern")`, `SRV("mw")=count`,
  `SRV("mw",n)=ref` (see `route`/`use^STDHTTPD`). A first guess of
  `SRV("route","GET","/healthz")` is wrong → scan the indexed nodes.
- **`parseRsp^STDHTTPMSG` preserves response header-name CASE** (unlike
  `parseReq`, which lowercases). A wire `WWW-Authenticate` parses back as
  `RSP("hdr","WWW-Authenticate")`, NOT lowercased — use the case-insensitive
  `$$hdr^STDHTTPMSG(.R,"www-authenticate")` for the challenge, never a raw
  `$get(R("hdr","www-authenticate"))`.
- **Byte mode is MANDATORY on YDB and the Makefile didn't encode it.** Without
  `--chset m`, **VWEBATST aborts 0/0** (its JWT/HMAC signing emits bytes >127 that
  re-encode under UTF-8). The suites always ran with `--chset m` by hand; the
  Makefile `test`/`coverage`/`check` targets passed no chset, so `make check` was
  not actually green for YDB as written. Fixed: a **`CHSET` Makefile var**
  defaulting to `m` for `ENGINE=ydb` (empty for IRIS), folded into `ENGINE_FLAGS`
  — mirrors m-stdlib's `M_ENGINE_FLAGS` byte-mode default. `make test ENGINE=ydb
  DOCKER=m-test-engine` now byte-mode by default.

## KIDS install→verify→back-out (Decision Q2 — documented, NOT run live)

The M0a lifecycle of `VWEB*1.0*3` over the driver is the close-out step, but a
real live install→back-out would **expose** the known v-pkg gap rather than
exercise it cleanly: **v-pkg uninstall is routine-only (#9.7/#9.6)**, so backing
out `VWEB*1.0*3` would **orphan the 9 XPAR param defs (#8989.51) + the `VWEB
LISTENER` option (#19)**. Per Q2, M6.6 **documents** the manual cleanup and files
the enhancement separately rather than mutating shared live VistA with a partial
back-out:
- **Manual back-out cleanup** (until v-pkg covers it): after the routine
  uninstall, delete the `VWEB *` parameter definitions from #8989.51 and the
  `VWEB LISTENER` entry from #19 (OPTION) by hand.
- **Enhancement owed (separate, cross-repo):** extend `v pkg uninstall` to cover
  #8989.51 + #19 (the M6.3/M6.4 residual). Not blocking the M6 capstone close.
