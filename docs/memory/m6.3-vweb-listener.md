# VSL/MSL M6.3 — `vweb`, the inbound socket adapter, 2026-06-17

The first **VistA-coupled** milestone of the M6 VWEB capstone: a real socket →
`STDHTTPD`'s portable worker loop → byte-exact response back, on both engines.
A NEW repo `v-web` (`~/vista-cloud-dev/v-web`), layer `v`, routine namespace
`VWEB*`. Branch `m6.3-vweb-listener`. Consumes m-stdlib (MSL **v0.12.2** since the 2026-06-17 repins:
`STDHTTPD`/`STDHTTPMSG`/`STDNET`) and v-stdlib (`VSLTASK`/`VSLCFG`/`VSLENV`)
upward — one-way `v→m`, the waterline holds.

## What shipped (4 routines + 4 suites + KIDS)

- **VWEBIO** — the engine device/TLS transport adapter. `transport(.TR)` builds
  the descriptor `$$serve` consumes (`TR("read")="read^VWEBIO"`,
  `TR("write")="write^VWEBIO"`); `$$read`/`$$write`/`$$listen`/`$$boundport`/
  `$$accept`/`$$close` are **thin signature-compatible pass-throughs to STDNET**
  (the only m-dep). `$$listenTls(port,config)` is **gap-loud** — raises
  `,U-VWEB-NOTLS,` (flag-`$ETRAP`, detail in `^TMP($job,"vwebio","err")`); never
  serves plaintext on the TLS port. `$$tlsAvailable()`=0 (M2.T2 blocked).
- **VWEBL** — the listener launcher + accept-loop + worker handoff (spec §4).
  `$$launch()` = `$$schedule^VSLTASK("run^VWEBL",…)` (the startup OPTION points
  here); `run()` opens the socket on the XPAR port and enters `acceptLoop`
  (serial accept→serve→close until `$$stop^VSLTASK`); `$$serveConn(.SRV,conn,
  .opts)` = `do transport^VWEBIO` + `$$serve^STDHTTPD` + `$$close^VWEBIO`;
  `$$acceptOne` = accept + serveConn; `healthRoutes(.SRV)` registers
  `GET /healthz` → `health` (200 "ok"). **No FileMan routes (M6.4), no auth
  (M6.5).**
- **VWEBCFG** — XPAR config accessors (`$$port` 8089 / `$$tlsConfig` "" /
  `$$idleTimeout` 30 / `$$maxBody` 1048576 / `$$maxWorkers` 1), **delegated to
  `$$get^VSLCFG`** (v→v). No direct `^XPAR` call → ICR registry stays empty.
- **VWEBENV** — the KIDS env-check; `check(.facts)` = `$$check^VSLENV` base facts
  (engine/version/kernel/tls) + a `facts("vweb","tls")` server-TLS-config flag.
  Delegates to resident VSL* (no direct Kernel/XPAR token).

## Design decisions (per the kickoff, resolved — do not re-litigate)

- **D1 socket handoff = single-process SERIAL** (accept→serve→close→accept, no
  JOB-off). The vertical is proven end-to-end first; the jobbed-worker
  concurrency (the engine-divergent `$zversion["IRIS"` fork) is a separately-
  tested LATER layer, not a blocker. `acceptOne` is the asserted serial unit.
- **HTTP-first, TLS-gap-loud** — the dev/test path is plaintext loopback; the TLS
  code path is bound + raises `,U-VWEB-NOTLS,`; live TLS verification is owed
  (M2.T2 infra-blocked, no cert / `XU*8.0*787`).
- **Zero direct VistA calls in `VWEB*`** — every XPAR/Kernel/TaskMan touch is
  delegated to a resident `VSL*` routine (`$$get^VSLCFG`, `$$check^VSLENV`,
  `$$schedule`/`$$stop^VSLTASK`). So the **ICR registry is empty** and the
  no-direct-global gate is trivially green — the cleanest possible waterline.

## Hard-won gotchas (durable)

1. **`do <value-returning-function>` aborts with `%YDB-E-NOTEXTRINSIC`** — the
   `$quit` gotcha. `close^VWEBIO` ends `quit $$close^STDNET(id)`; calling it as
   `do close^VWEBIO(conn)` (in serveConn/run) faulted *uncatchably* → the suite
   aborted to **0/0** (report never fired). Fix: call it as `$$` (`set ok=$$close^VWEBIO(conn)`).
   This is the same family as STDHTTPD's gotcha-3, but the trap-presence masked
   it: a value-fn fault one frame below a parent `$etrap` *resumes* the parent;
   inline (same frame) it unwinds. Lesson: **call every `$$` wrapper with `$$`,
   never `do`.**
2. **A bare m-test-engine has no Kernel** — `$$GET^XPAR`/`$$VERSION^XPDUTL`/
   `$$TM^%ZTLOAD` are missing, and a *missing-routine fault on these does NOT
   resume* a flag-`$ETRAP` cleanly (unlike a synthetic missing routine — engine
   nuance), so it aborts the suite. **Guard with `$text(GET^XPAR)`/
   `$text(VERSION^XPDUTL)`/`$text(TM^%ZTLOAD)`** before the call (in the TEST,
   which is not ICR-scanned) and soft-skip — never trap it.
3. **The test harness hides assertion messages** (json/text show only counts).
   To debug an engine fault: stash `$zstatus` to a scratch global inside the
   routine's own `$etrap`, then read it back with
   `m vista exec --engine … --transport docker --container … --namespace … 'write …'`
   (driver-compliant; works on bare engines too). This is how the NOTEXTRINSIC
   and the IRIS-CRLF defects were pinned.
