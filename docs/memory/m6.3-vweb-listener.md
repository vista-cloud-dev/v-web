## Design decisions (per the kickoff, resolved â€” do not re-litigate)

- **D1 socket handoff = single-process SERIAL** (acceptâ†’serveâ†’closeâ†’accept, no
  JOB-off). The vertical is proven end-to-end first; the jobbed-worker
  concurrency (the engine-divergent `$zversion["IRIS"` fork) is a separately-
  tested LATER layer, not a blocker. `acceptOne` is the asserted serial unit.
- **HTTP-first, TLS-gap-loud** â€” the dev/test path is plaintext loopback; the TLS
  code path is bound + raises `,U-VWEB-NOTLS,`; live TLS verification is owed
  (M2.T2 infra-blocked, no cert / `XU*8.0*787`).
- **Zero direct VistA calls in `VWEB*`** â€” every XPAR/Kernel/TaskMan touch is
  delegated to a resident `VSL*` routine (`$$get^VSLCFG`, `$$check^VSLENV`,
  `$$schedule`/`$$stop^VSLTASK`). So the **ICR registry is empty** and the
  no-direct-global gate is trivially green â€” the cleanest possible waterline.

## Hard-won gotchas (durable)

1. **`do <value-returning-function>` aborts with `%YDB-E-NOTEXTRINSIC`** â€” the
   `$quit` gotcha. `close^VWEBIO` ends `quit $$close^STDNET(id)`; calling it as
   `do close^VWEBIO(conn)` (in serveConn/run) faulted *uncatchably* â†’ the suite
   aborted to **0/0** (report never fired). Fix: call it as `$$` (`set ok=$$close^VWEBIO(conn)`).
   This is the same family as STDHTTPD's gotcha-3, but the trap-presence masked
   it: a value-fn fault one frame below a parent `$etrap` *resumes* the parent;
   inline (same frame) it unwinds. Lesson: **call every `$$` wrapper with `$$`,
   never `do`.**
2. **A bare m-test-engine has no Kernel** â€” `$$GET^XPAR`/`$$VERSION^XPDUTL`/
   `$$TM^%ZTLOAD` are missing, and a *missing-routine fault on these does NOT
   resume* a flag-`$ETRAP` cleanly (unlike a synthetic missing routine â€” engine
   nuance), so it aborts the suite. **Guard with `$text(GET^XPAR)`/
   `$text(VERSION^XPDUTL)`/`$text(TM^%ZTLOAD)`** before the call (in the TEST,
   which is not ICR-scanned) and soft-skip â€” never trap it.
3. **The test harness hides assertion messages** (json/text show only counts).
   To debug an engine fault: stash `$zstatus` to a scratch global inside the
   routine's own `$etrap`, then read it back with
   `m vista exec --engine â€¦ --transport docker --container â€¦ --namespace â€¦ 'write â€¦'`
   (driver-compliant; works on bare engines too). This is how the NOTEXTRINSIC
   and the IRIS-CRLF defects were pinned.
